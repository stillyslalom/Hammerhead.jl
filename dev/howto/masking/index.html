Mask reflections and geometry

Goal: exclude a region — model geometry, a wall, a persistent reflection — from the analysis so it produces no vectors and cannot contaminate its neighbors. Background on the semantics is in the masking model.

Draw a polygon mask

polygon_mask rasterizes a polygon given as (x, y) corners in pixel coordinates (x along columns):

using Hammerhead

mask = polygon_mask(size(imgA), [(120, 40), (480, 90), (450, 300), (100, 260)])
result = run_piv(imgA, imgB, passes; mask)

Combine several exclusion regions with element-wise OR:

blade  = polygon_mask(size(imgA), blade_vertices)
glare  = polygon_mask(size(imgA), glare_vertices)
result = run_piv(imgA, imgB, passes; mask = blade .| glare)

Load a mask image

Many datasets ship a mask as an image file (e.g. the PIV Challenge case 1C impeller mask). load_mask thresholds it into a Bool matrix:

mask = load_mask("mask.png")                 # bright pixels = excluded
mask = load_mask("mask.png"; invert = true)  # dark pixels = excluded

Any image-sized Bool/BitMatrix works as a mask — build one from raw pixel logic if that's easier, e.g. mask = background .> 0.8 to exclude persistently bright (reflective) pixels of a compute_background image.

Control when a window is dropped

Windows whose masked-pixel fraction reaches mask_threshold (default 0.5) are dropped entirely; windows below it correlate over their valid pixels only. Lower the threshold to be more conservative near edges:

result = run_piv(imgA, imgB, passes; mask, mask_threshold = 0.25)

Read the results correctly

result.mask       # true = window dropped (NaN fields, no measurement)
result.outliers   # true = measured but failed validation (replaced if enabled)
valid = .!(result.mask .| result.outliers)

Masked cells are never counted as outliers, and NaNs in u/v at masked cells are intentional — filter with valid before computing your own statistics (the built-in field_statistics and error_statistics already do).

Batch and stereo

The same mask keyword flows through run_piv_sequence and run_piv_ensemble (one static mask for the whole sequence). For stereo, pass a grid-sized mask to run_piv_stereo / self_calibrate; it is combined with the dewarpers' out-of-view masks automatically.
